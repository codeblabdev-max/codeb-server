## 🤖 자동 기록 예시

### 예시: 사용자 인증 플로우 통합 테스트
```markdown
## 통합 테스트 - 사용자 인증 플로우
**생성 시점**: 2024-09-04T15:45:00Z  
**Context 정보**:
- 관련 컴포넌트: LoginForm, AuthAPI, UserSession, Dashboard
- 관련 API: POST /api/auth/login, GET /api/auth/profile
- 관련 DB: users, sessions 테이블

### E2E 테스트 시나리오
#### 성공적인 로그인 플로우
1. [ ] 로그인 페이지 접속
2. [ ] 유효한 이메일/비밀번호 입력
3. [ ] 로그인 버튼 클릭
4. [ ] API 요청 확인 (POST /api/auth/login)
5. [ ] JWT 토큰 저장 확인
6. [ ] 대시보드 페이지로 리다이렉트
7. [ ] 사용자 정보 표시 확인

#### 실패 케이스 처리
1. [ ] 잘못된 이메일 형식
2. [ ] 존재하지 않는 사용자
3. [ ] 잘못된 비밀번호
4. [ ] 네트워크 오류 처리
5. [ ] 토큰 만료 처리

#### 보안 테스트
- [ ] XSS 공격 시도
- [ ] CSRF 토큰 검증
- [ ] SQL Injection 방어
- [ ] 브루트포스 공격 방어
```

```markdown
### E2E 테스트 시나리오  
#### 완전한 구매 플로우
1. [ ] 상품 목록 페이지 접속
2. [ ] 상품 검색 및 필터링
3. [ ] 상품 상세 페이지 확인
4. [ ] 장바구니 추가
5. [ ] 장바구니 페이지에서 수량 조정
6. [ ] 결제 페이지 이동
7. [ ] 배송 정보 입력
8. [ ] 결제 정보 입력
9. [ ] 주문 완료 확인
10. [ ] 주문 확인 이메일 발송

#### 예외 상황 테스트
- [ ] 재고 부족 시 처리
- [ ] 결제 실패 시 복구
- [ ] 세션 만료 중 주문 진행
- [ ] 동시 주문 시 재고 처리
- [ ] 취소/환불 프로세스

#### 성능 테스트
- [ ] 대용량 주문 처리
- [ ] 동시 접속자 처리
- [ ] 결제 게이트웨이 응답 시간
```

## 🔄 업데이트 규칙

### 자동 추가 조건
1. **완전한 기능 구현**: UI + API + DB가 모두 완성된 기능
2. **크리티컬 패스**: 핵심 비즈니스 로직이 포함된 워크플로우
3. **외부 연동**: 결제, 이메일, SMS 등 외부 서비스 통합
4. **보안 중요**: 인증, 권한, 개인정보 처리 관련 기능

### 기록 정보
- **플로우 맵**: 사용자 여정과 시스템 상호작용
- **의존성**: 관련 컴포넌트, API, 데이터베이스
- **테스트 데이터**: 시나리오별 필요한 테스트 데이터
- **성능 기준**: 응답 시간, 처리량, 리소스 사용량
