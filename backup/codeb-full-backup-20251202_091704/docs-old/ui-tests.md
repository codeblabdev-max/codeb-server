# UI 컴포넌트 테스트 항목

## 🎯 개요
Context가 유지된 상태에서 자동으로 기록되는 UI 컴포넌트 테스트 항목들

## 📋 현재 등록된 UI 테스트

### ✅ 완료된 테스트
- 없음 (신규 프로젝트)

### 📝 테스트 대기 목록
- 페이지/컴포넌트 생성 시 자동 추가됩니다

## 🤖 자동 기록 예시

### 예시: LoginPage 컴포넌트 생성 시
```markdown
## 테스트 항목 - LoginPage  
**생성 시점**: 2024-09-04T15:45:00Z  
**Context 정보**: 
- 클래스명: `.login-form`, `.email-input`, `.password-input`
- ID: `#login-btn`, `#forgot-password-link`  
- 함수: `handleLogin()`, `validateForm()`

### 테스트 케이스
- [ ] 로그인 폼 렌더링 확인
- [ ] 이메일 입력 필드 (.email-input) 동작 확인
- [ ] 비밀번호 입력 필드 (.password-input) 동작 확인
- [ ] 로그인 버튼 클릭 (#login-btn) 이벤트 처리
- [ ] 유효성 검사 오류 메시지 표시
- [ ] 성공 시 대시보드 페이지로 리다이렉트
- [ ] 실패 시 오류 메시지 표시
- [ ] "비밀번호 찾기" 링크 (#forgot-password-link) 동작
```

## 🔄 업데이트 규칙

### 자동 추가 조건
1. **새 페이지 생성**: React/Next.js 페이지 컴포넌트 생성 시
2. **컴포넌트 생성**: 재사용 가능한 UI 컴포넌트 생성 시  
3. **스타일 추가**: CSS 클래스나 ID 속성 추가 시
4. **이벤트 핸들러**: 사용자 상호작용 함수 추가 시

### 기록 정보
- **Context 정보**: 클래스명, ID, 데이터 속성
- **이벤트 핸들러**: onClick, onSubmit, onChange 등
- **상태 관리**: useState, useEffect, 상태 변화
- **Props/데이터**: 전달되는 데이터 구조

- **컴포넌트 테스트**: 0개
- **상호작용 테스트**: 0개
